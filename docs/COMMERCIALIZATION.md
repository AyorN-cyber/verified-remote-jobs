# Commercialization Options

This project can remain open-source while still leaving room for paid products.

## Option 1: Open Core

Keep the verifier, source connectors, and exports open-source. Sell hosted features:

- scheduled scans
- email alerts
- team dashboards
- richer ATS connectors
- verified company contact enrichment
- managed API keys
- higher-volume queues

## Option 2: Hosted SaaS

Offer a web app where users bring a CV, choose roles/countries, and receive verified prospects. Charge by:

- monthly subscription
- credits per verification run
- team seats
- pay-as-you-go export packs

## Option 3: Services

Sell done-for-you verified job prospect packs for job seekers, recruiters, career coaches, and bootcamps. This is easier to start than SaaS because it does not require billing, accounts, and multi-user infrastructure on day one.

## Licensing Note

Pick the license intentionally before publishing:

- MIT/Apache-2.0: easiest adoption, weakest commercial control.
- AGPL-3.0: stronger pressure for hosted forks to share modifications.
- Business Source License or dual license: more commercial control, but less open-source friendly.

For a public portfolio and adoption, MIT or Apache-2.0 is simpler. For selling a hosted product later, keep API keys, hosted scheduling, contact enrichment, and premium connectors outside the open-source core.
